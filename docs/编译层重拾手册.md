## 三、2026-06-10 踩坑记录（重要）

### 问题现象

部署后通过 `zs.captainxq.me` 访问，浏览页 tab 多了"编译论点"，但硬知识/软智慧点开后一片空白。本地 curl 验证 HTML 内容完整。

### 根因分析

**不是代码问题，是部署验证流程的缺口：**

1. **操盘手改完 web_server.py 后没重启 hermes-web 服务**
   - Flask 还在跑旧代码
   - 新代码根本没生效
2. **操盘手没在本地 curl 验证**
   - 没 `curl http://localhost:5000/browse` 确认返回正确 HTML
   - 直接通过 Cloudflare 域名验证，中间隔了缓存
3. **Cloudflare 缓存了旧 HTML 页面**
   - browse 页面没有 `Cache-Control: no-cache` 头
   - 用户浏览器也缓存了旧页面
   - 导致看到的是"残影"——tab 变多了但内容对不上

### 教训总结

涉及 web_server.py 或 build_static.py 的改动后，验证流程必须严格按顺序：

```
改代码 → 重启 hermes-web → curl localhost:5000/browse 确认
  → 浏览器开无痕窗口验证 → 再通过公网域名验证
```

不能跳过重启步骤，不能通过 Cloudflare 域名直接验证（会被缓存骗）。

### 无痕窗口提示

Cloudflare Tunnel 的网站可能被 Cloudflare 边缘缓存 HTML。如果改了前端渲染逻辑，建议用浏览器**无痕/隐私窗口**访问公网域名，跳过本地浏览器缓存。
