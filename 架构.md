# Hermes 架构

## 技术栈

| 层 | 选型 | 说明 |
|----|------|------|
| 语言 | Python 3.13 | 本地运行，无外部依赖框架 |
| AI | DeepSeek API (deepseek-chat) | 分类、提纯、问题生成、问答 |
| 同步 | Syncthing v2.1.0 | 手机 ↔ 电脑 原子文件同步 |
| 索引 | JSON 文件 + SHA256 | 轻量增量索引，无数据库 |
| Web 服务 | Flask | 本地 HTTP 服务，响应手机端请求 |
| 语义检索 | sentence-transformers (bge-small-zh) | 本地嵌入模型，零成本、无 API 依赖 |
| 穿透 | Cloudflare Tunnel | 本地服务穿透到公网，手机浏览器访问 |

## 模块划分

```
┌─────────────────────────────────────┐
│            main.py（管线调度）        │
├──────┬──────┬──────────┬────────────┤
│Bootstrap│Engine│Index Guard│Weekly Scan│
│ 启动自检 │ 分流 │ 增量索引  │  周度审计  │
└──────┴──────┴──────────┴────────────┘
```

| 模块 | 文件 | 职责 | 阻断性 |
|------|------|------|--------|
| 调度器 | `main.py` | 管线编排，JSON 最终报告 | — |
| 自检 | `scripts/bootstrap.py` | 目录/文件存在性、Syncthing 守护 | 阻断 |
| 引擎 | `scripts/hermes_engine.py` | 三阶段事务分流 + AI 分类/提纯/分裂 | 非阻断 |
| 索引 | `scripts/index_guard.py` | SHA256 + mtime 增量索引 | 非阻断 |
| 周扫 | `scripts/weekly_scan.py` | 摘要审计 + Dashboard 问题队列 | 非阻断 |
| Web 服务 | `api/web_server.py` | Flask 服务，提供问答接口和知识浏览 | 非阻断 |
| 问答引擎 | `api/qa_engine.py` | Jina 检索 + DeepSeek 生成 | 非阻断 |
| 语义索引 | `api/semantic_index.py` | 管理向量索引，增量更新 | 非阻断 |

## 核心数据流

```
手机采集（Markor 原子文件 *.md）
  → Syncthing 同步
  → kb/inbox/
  → Hermes Engine: Move → Process（AI分类+提纯+URL抓取+背景注入）→ Commit
  → core/（认知资产）│ manual/（技术知识）│ raw/（原文冷备）
  → Index Guard（增量索引）
  → Weekly Scan（摘要审计 → 生成问题 → Dashboard Active/Backlog）
  → 000_Dashboard.md（用户可见）
  → 用户回答 → 新 Core 笔记 → 进入下一轮
```

## 知识分流规则

### 顶层分类：硬知识 / 软智慧

AI 首先判断 `knowledge_type`：

| 类型 | 定义 | 路由 |
|------|------|------|
| hard | 工具、API、命令、代码、操作步骤 | `manual/technical/` |
| soft | 商业判断、方法论、投资理念、认知升级 | 见下方拆分 |
| mixed | 同时包含硬知识和软智慧 | 细胞分裂 + 双写 |

### soft 子级拆分（按认知分）

| 条件 | 目标 |
|------|------|
| soft + cog ≥ 3 | `core/insight/` |
| soft + cog < 3 | `core/note/` |

### mixed 双写规则

| 条件 | 目标 |
|------|------|
| mixed + cog ≥ 5 → insight 端 | `core/insight/` |
| mixed + tech ≥ 5 → tech 端 | `manual/technical/` |

## 关键设计原则

| 原则 | 实现 |
|------|------|
| 事务安全 | Move → Process → Commit 三段式 |
| 可恢复 | 失败文件保留 processing，死信队列上限 3 次 |
| 原文保真 | raw/ 保存原始文件 + URL 抓取冷备 |
| 防爆设计 | 摘要级周扫（只读 `_refined.md`），1000 篇规模安全 |
| AI 只提问 | 周扫输出的是问题，不是总结 |
| 增量索引 | SHA256 对比，只更新变更文件 |
| 双策略编码 | hard用技术骨架（定位/方法/避坑），soft用认知骨架（洞察/逻辑链/金句） |

## 文件命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 手机采集 | `clip_YYYYMMDD_HHMMSS.md` | `clip_20260528_224512.md` |
| 提纯版 | `{原名}_refined.md` | `clip_20260528_224512_refined.md` |
| 技术分裂 | `tech_{原名}.md` | `tech_clip_test_mixed.md` |
| 认知分裂 | `insight_{原名}.md` | `insight_clip_test_mixed.md` |
| URL 冷备 | `{原名}_source_{hash8}.md` | `clip_test_mixed_source_a1b2c3d4.md` |
| 周扫问题 | `question_YYYYMMDD_HHMMSS_{n}.md` | `question_20260528_212715_1.md` |
