💡 关联底层认知思考：[[insight_prompt]]

### 技术实现细节

#### 万能Prompt公式
- 公式：角色 + 任务 + 背景 + 格式 + 限制
- 示例：
  - 角色：你是一位资深产品经理
  - 任务：帮我分析这个功能需求的可行性
  - 背景：我们是一家B2B SaaS公司，目标用户是中小企业
  - 格式：用结构化列表呈现，分「优势」「风险」「建议」三块
  - 限制：300字以内，不需要写结论
- 组合后Prompt：
  「你是一位资深产品经理。帮我分析这个功能需求的可行性：用户希望在App里直接发起视频会议。我们是一家B2B SaaS公司，目标用户是中小企业。请用结构化列表呈现，分「优势」「风险」「建议」三块，300字以内。」

#### 进阶技巧
- **说原因**：
  - ❌ 「不要用省略号。」
  - ✅ 「不要用省略号，因为这段文字会被语音播报系统朗读，省略号会导致语音引擎不知道怎么处理。」
- **给示例**：
  - 示例：
    「2026.04 更新
    新增：支持批量导出PDF，最多50个文件一次搞定
    优化：搜索速度提升40%，大文件也不卡了
    修复：偶尔登录失败的问题」
  - 指令：现在请按这个风格，帮我写以下功能的更新日志：[粘贴功能列表]
- **分步思考**：
  - ❌ 「我应该跳槽吗？」
  - ✅ 「我在考虑是否跳槽，请一步步帮我分析：
    1. 先列出跳槽的潜在收益
    2. 再列出潜在风险和代价
    3. 分析我当前情况下哪些因素权重最高
    4. 最后给出你的建议和理由
    我的情况：目前在一家中型互联网公司做产品经理，3年经验，月薪2万，新机会是一家创业公司，月薪3万但没有期权。」
- **长内容前置**：
  - 正确做法：[粘贴长文档] → 请根据以上内容，总结3个核心结论
  - 错误做法：帮我总结以下内容的3个核心结论 → [粘贴长文档]
- **指定格式**：
  - 表格：列包含：竞品名称 / 核心优势 / 价格 / 适合场景
  - 分级结构：按重要性从高到低排列
  - 待办清单：具体动作 + 截止时间 + 预期产出
  - 固定模板：方案名称 / 优点 / 缺点 / 适合场景 / 我的建议
- **排除指令**：
  - 「不要写开头的寒暄和结尾的客套」
  - 「不要用『赋能』『闭环』『颗粒度』这类词」
  - 「不要超过5条，宁少勿多」
  - 「不要解释原因，直接给结论」
  - 「不要加emoji」

#### 高级用法
- **Prompt链**：
  1. 选题Prompt：请给我5个不同角度的选题方向
  2. 大纲Prompt：就第3个方向，展开6-8个小节的大纲
  3. 逐节撰写Prompt：按大纲第1节，写200-300字
  4. 整合优化Prompt：通读所有小节，检查逻辑和风格
- **自检反驳**：
  - 指令：「你刚才给了我方案A的分析，现在请站在反对者的角度，列出方案A最大的3个风险和可能失败的原因。」
- **多角色视角**：
  - 指令：「请分别从产品经理、法务、用户运营三个角色分析这个功能。」
- **元提示**：
  - 指令：「我想让AI帮我分析竞品，但我不知道怎么写Prompt。请帮我设计一个专业的竞品分析Prompt。」
- **角色反转**：
  - 指令：「请扮演一位顶级创业导师，通过提问来帮我厘清方向。先问我第一个最关键的问题，等我回答后，再问下一个。」
- **给AI压力**：
  - 指令：「这个答案太表面了。请假设你是领域内最顶尖的专家，给出一个普通人想不到的深度见解。」

#### 迭代方法
- 不要这样说：「不好，重写」
- 要这样说：「第二段太正式了，语气改成跟朋友聊天的感觉」
- 不要这样说：「不是我想要的」
- 要这样说：「方向对了，但缺少具体数据支撑，每个结论加一个真实案例」
- 不要这样说：「再试试」
- 要这样说：「整体结构保留，但结尾太弱了，换一个有行动号召的结尾」
