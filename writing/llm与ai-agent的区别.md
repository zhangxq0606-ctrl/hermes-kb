# LLM 和 AI Agent 有什么区别？一句话说清，再深入一层

你有没有这种感觉：

打开一篇讲 AI Agent 的文章，看完觉得"懂了"。关掉之后别人问你 Agent 和 ChatGPT 到底有啥区别，你又说不清楚。

不是你的问题。大部分文章只给了你一个比喻——"Agent 是有手有脚的大脑"——但没告诉你手脚长什么样、怎么动起来的。

今天分两层。第一层让你能跟朋友说明白，第二层让你真的搞懂。

---

## 01 第一层：一句话说清

**LLM 是超级学霸。** 你问它答，它不说它不做。

你让它"帮我写一封请假邮件"，它直接给你写出来。你让它"查一下这周的天气再帮我约个饭局"，它会告诉你天气，然后让你自己去约。不是不想帮你，是没手。

**Agent 不一样。Agent = LLM + 记忆 + 工具 + 规划能力。**

你让 Agent "帮我找北京月薪 3 万以上的产品经理岗位，整理成表格发我"，它自己联网搜索、筛选、整理、发邮件。你不用一步步教它。

> 一句话：LLM 是说的，Agent 是做的。

---

## 02 第二层：三个核心差异

只停在这里，你跟别人聊的时候还是会露馅。因为"有手有脚"只是个比喻，真正区别在架构。拆成三个维度你就清楚了。

### 维度一：被动 vs 主动

LLM 是问答机。你给一段输入，它生成一段输出。没有指令它就待机，对话框一关什么都不记得。

Agent 有**执行循环**。收到一个目标后，自己拆步骤、调工具、检查结果，不对就重来。整个过程不需要你每一步都插手。

就像你的 Codex——你丢一个需求过去，它自己翻代码、改文件、跑测试。你不是在跟它"聊天"，是在给它派活。

### 维度二：无工具 vs 调工具

这是两者最实质的差别。

LLM 只有一个能力——生成文字。它没法查日历、没法打开网页、没法操作你的文件。

Agent 带一个**工具箱**。需要查信息就调搜索，需要算数据就调代码解释器，需要写文件就调文件操作。工具越多，Agent 能做的事就越多。

这也是为什么 MCP 很关键——它让 Agent 的工具箱变成了即插即用的 USB 接口。

### 维度三：无记忆 vs 有记忆

你跟 ChatGPT 聊完关掉，下次打开它不认识你。因为这个 session 结束了。

Agent 有**长期记忆**。能记住你上周提过的需求、你做事的偏好、你之前拒绝过什么方案。不是因为它天生"记得"，是因为背后挂了向量数据库或外部存储。

---

## 03 所以回到开头的问题

LLM 和 Agent 不是同一个东西的升级版。它们是两个层级的东西——LLM 是能力，Agent 是系统。

LLM 是 Agent 的大脑，但 Agent 不止有大脑。它多了三层东西：**能自己规划、能调用工具、能记住你。**

现在你知道了区别。下一篇我们聊另一个问题：**是不是所有事都应该上 Agent？** 答案可能跟你想的不一样。
