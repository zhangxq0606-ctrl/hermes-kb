# 上传指南

> 这是一份给 AI 代理的完整指南。收到后请按顺序执行，从理解项目到最终部署上线。

---

## 第一步：理解这个网站

请先读取以下文件，了解项目全貌：

1. `架构.md` —— 技术栈、目录结构、路由设计、核心数据流
2. `进度.md` —— 当前项目阶段、已完成和待办

## 第二步：了解内容格式

### 文件位置

| 内容类型 | 目录 |
|----------|------|
| 项目 | `src/content/projects/<slug>.md` |
| 复盘/博客 | `src/content/blog/<slug>.md` |

`<slug>` 用英文短横线命名，如 `carpooling.md`、`carpooling-retro.md`。不要中文文件名。

### 项目 Frontmatter

```yaml
---
title: "项目名称"
date: 2026-06-03
status: "进行中"          # 三选一：进行中 | 已完成 | 暂停
tags: ["标签1", "标签2"]    # 用于筛选和展示
summary: "一句话说明项目要解决什么问题"
description: "可选，SEO 摘要"  # 建议加
stack: ["技术1", "技术2"]   # 可选
links:                     # 可选，没有就整段删掉
  github: "https://..."
  demo: "https://..."
---
```

### 博客/复盘 Frontmatter

```yaml
---
title: "文章标题"
date: 2026-06-03
description: "可选，显示在博客列表页的摘要"  # 建议加
project: carpooling         # 可选，填项目的 slug，实现双向关联
tags: ["产品决策", "复盘"]   # 可选
---
```

### 正文规范

项目页覆盖四个区域：
1. **背景** — 为什么做，解决什么问题
2. **核心决策** — 关键选择及理由
3. **架构概览** — 一段话 + 可选简图
4. **当前状态** — 进行到哪了，下一步是什么

博客/复盘没有固定格式，围绕一个具体决策点或阶段写反思即可。

### 示例模板

写内容前先看 `src/content/projects/personal-website.md` 或 `src/content/blog/` 下已有的文件，直接复制一个作为模板。

### 已知问题

- Astro 5.18 的 `entry.id` 包含 `.md` 后缀，拼接路径时记得 `.replace(/\.md$/, '')`

## 第三步：编写内容

根据用户提供的项目/博客描述，在对应目录下创建 `.md` 文件。

## 第四步：部署上线

内容写完后，执行以下命令：

```bash
git add .
git commit -m "内容更新：[项目/文章名称]"
git push
```
